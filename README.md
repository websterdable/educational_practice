# ChemAI: Predict the Cure — ML для предсказания противовирусной активности

## О проекте

Хакатон по разработке ML-модели для предсказания эффективности химических соединений против вируса гриппа. 

Задача: по 214 молекулярным дескрипторам предсказать три показателя — IC50 (активность), CC50 (токсичность), SI (индекс селективности). 

Метрика: среднее RMSE по трём таргетам.

## Данные

**Исходные данные:** 751 молекула (train) / 250 (test), 214 дескрипторов  + 3 целевые переменные.

**Анализ и предобработка:**
- Обнаружено, что `SI = CC50 / IC50`. Однако прямое вычисление SI может накапливать ошибки предсказаний IC50 и CC50, поэтому тестировались два подхода: предсказание двух таргетов с вычислением SI и предсказание всех трёх.
- Распределения таргетов скошены вправо — применено `sqrt`-преобразование (мягче, чем `log`, лучше сохраняет информацию в области малых значений).
- Признаки разделены на смысловые группы (общие свойства, электронные, топологические, VSA, функциональные группы, специфические дескрипторы). Анализ мультиколлинеарности и удаление избыточных признаков проводились **внутри каждой группы** с учётом:
  - Химической иерархии (специфичный признак -> общий)
  - Биологической интерпретируемости
  - Корреляций (удалялись только почти полные дубликаты с r > 0.98 и структурно избыточные)
- Создано 5 новых гибридных дескрипторов: плотность массы, удельная гидрофобность поверхности, доля ароматических колец, электронная насыщенность, топологическая сложность.

**Результат:** 4 итоговых CSV-файла в `data/processed/`:
- `train_final.csv` / `test_final.csv` — после полной очистки и отбора
- `train_with_engineered_features.csv` / `test_with_engineered_features.csv` — с добавленными признаками

## Модели

### Подход к моделированию

Для каждой конфигурации (2 датасета × 2 варианта таргетов) обучены модели с 5-фолдовой кросс-валидацией. Метрика — `custom_rmse` (среднее RMSE по трём таргетам в исходных единицах mM).

**Почему эти модели:**
- Данные нелинейны (корреляции признаков с таргетами < 0.3)
- Бустинги устойчивы к мультиколлинеарности
- CatBoost и LightGBM нативно обрабатывают NaN (отсутствие признака в молекуле — тоже информация, заполнение средним/медианой противоречит химической логике)

### Базовые модели

| Модель | Назначение |
|--------|------------|
| **Random Forest** | Бейзлайн. Выбран вместо линейной регрессии, т.к. данные нелинейны |
| **CatBoost** | Симметричные деревья, устойчив к переобучению |
| **LightGBM** | Листовые деревья, быстрая сходимость |
| **XGBoost** | Классический градиентный бустинг с регуляризацией |

### Ансамбли

Построены 4 комбинации: CB+XGB, LGB+XGB, CB+LGB, CB+LGB+XGB. Подбор весов — перебор на 3-фолдовой CV. Результат: ансамбли не превзошли одиночные модели.

### Оптимизация гиперпараметров

Для лучших моделей (CatBoost и LightGBM) выполнен подбор гиперпараметров через Optuna: 30 trials, 3-фолдовая CV. Подбирались: глубина деревьев, learning rate, L2-регуляризация, subsample.

### Итоговый результат

**Лучшая модель:** LightGBM с Optuna-параметрами, датасет `final`, 3 таргета.
